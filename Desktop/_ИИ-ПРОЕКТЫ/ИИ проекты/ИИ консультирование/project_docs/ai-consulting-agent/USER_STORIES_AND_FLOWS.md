## User Stories

### Story 1: Консультант Дмитрий входит в систему
```
Как консультант,
Я хочу безопасно войти в систему,
Чтобы только я видел свои проекты и данные клиентов.
```

### Story 2: Создание нового проекта
```
Как консультант,
Я хочу создать новый проект для клиента,
Чтобы начать процесс диагностики.
```

### Story 3: Запуск диагностического чата
```
Как консультант,
Я хочу пошагово провести диагностику через чат с AI,
Чтобы собрать необходимые данные о проблеме клиента.
```

### Story 4: Просмотр гипотез
```
Как консультант,
Я хочу увидеть гипотезы AI,
И выбрать самую сильную для отчёта.
```

### Story 5: Просмотр IQD (Data Quality)
```
Как консультант,
Я хочу видеть качество собранных данных (0-100%),
Чтобы понимать надёжность выводов.
```

### Story 6: Управление артефактами
```
Как консультант,
Я хочу загружать документы, видеть preview, удалять,
И организовывать материалы проекта.
```

### Story 7: Генерация отчёта
```
Как консультант,
Я хочу сгенерировать PDF отчёт для клиента,
С выводами, рекомендациями и источниками (background job).
```

### Story 8: Поделиться проектом
```
Как консультант Дмитрий,
Я хочу добавить коллегу Марию в проект,
Чтобы она помогала собирать данные.
```

### Story 9: Добавить артефакт в общий проект
```
Как Мария (junior),
Я хочу загрузить интервью в проект Дмитрия,
Чтобы помочь AI собрать больше данных.
```

### Story 10: Настройка AI параметров
```
Как администратор,
Я хочу изменить параметры AI (модель, температуру, знания),
Чтобы адаптировать поведение для разных типов проектов.
```

## User Journey: Газпром (3-дневный цикл)

### День 1 (Понедельник)
1. 09:00 — Дмитрий входит в систему
2. 09:15 — Создаёт проект "Gazprom Digital Transformation"
3. 09:30 — Начинает диагностику через чат (SSE streaming)
4. 10:00 — Загружает артефакты (PDF, XLSX, TXT)
5. 11:00 — Проверяет IQD (68%) и гипотезы (Score 87, 64, ...)
6. 11:30 — Запускает batch verify гипотез

### День 2 (Вторник)
1. 09:30 — Мария присоединяется к проекту
2. 10:00 — Добавляет новый интервью (IQD → 75%)
3. 11:00 — Дмитрий генерирует PDF отчёт (job queue, 5 мин)
4. 14:00 — Отправляет отчёт клиенту

### День 3 (Среда)
1. 10:00 — Встреча с клиентом, обратная связь
2. 15:00 — Обновляет отчёт (v2), удаляет Hypothesis 3
3. 16:00 — Архивирует проект
